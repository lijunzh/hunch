# Implementation Plan: #52 + #53 — Context-Based Episode & Year Resolution

> Replace fragile heuristics with cross-file invariance analysis.

## Problem Statement

Two heuristics violate P3 (Dumb engine, smart context):

1. **Digit decomposition** (`episodes/mod.rs` `try_digit_decomposition`)
   - Guesses season/episode from bare numbers by digit splitting: `501` → S5E01
   - Uses `filename.starts_with('[')` to detect anime (fragile)

2. **Year disambiguation** (`zone_map.rs` `disambiguate_years`)
   - Uses "last candidate = year, earlier = title" positional rule
   - Works for common cases but is fundamentally a guess

Both can be solved with the same tool: **variant/invariant analysis** across
sibling files.

## Key Insight

The existing `context.rs` only detects **invariant text** for title extraction.
We need to generalize it to detect **variant numbers** — numbers that change
across siblings are metadata (episode, year), numbers that don't change are
title content.

```
Siblings:                             Invariant   Variant
2001.A.Space.Odyssey.1968.1080p.mkv   "2001"      —
2001.A.Space.Odyssey.1968.720p.mkv    "2001"      —
→ 2001 is title (invariant), 1968 is year (invariant but structural position)

Show.501.720p.mkv                     —           501
Show.502.720p.mkv                     —           502
→ 501/502 are episodes (variant, sequential)

[Group] Title 03 (1080p).mkv          —           03
[Group] Title 04 (1080p).mkv          —           04
→ 03/04 are absolute episodes (variant, sequential)
```

## Architecture

### Current flow (no context)

```
Input → Pass1(tokenize → zone_map → match → resolve) → Pass2(rg, title, ep_title) → Result
                  ↑                                                ↑
           disambiguate_years()                         try_digit_decomposition()
           (positional heuristic)                       (digit-split heuristic)
```

### New flow (with context)

```
Target + Siblings → Pass1 on each → ContextAnalysis → Pass2 with signals → Result
                                         ↓
                                  ┌──────────────────┐
                                  │ InvarianceReport │
                                  │  .title: String  │
                                  │  .year_signals   │ → feeds zone_map year override
                                  │  .episode_signals│ → feeds episode extraction
                                  └──────────────────┘
```

### Without context (fallback)

Existing heuristics still run but emit `Confidence::Low`. No behavior change
for single-file parsing.

## Implementation Phases

### Phase 1: `InvarianceReport` struct + number analysis in `context.rs`

**Files:** `src/pipeline/context.rs`

Extend `context.rs` to analyze numbers across siblings:

```rust
/// Cross-file analysis results.
pub(crate) struct InvarianceReport {
    /// Invariant title text (existing functionality).
    pub title: Option<String>,
    /// Year signals: which year-like numbers are invariant (title) vs variant.
    pub year_signals: Vec<YearSignal>,
    /// Episode signals: which bare numbers form sequences across siblings.
    pub episode_signals: Vec<EpisodeSignal>,
}

pub(crate) struct YearSignal {
    /// Byte offset in the target file.
    pub position: usize,
    /// The 4-digit value.
    pub value: u32,
    /// Whether this number is the same across all siblings at this position.
    pub is_invariant: bool,  // invariant → title content, variant → metadata
}

pub(crate) struct EpisodeSignal {
    /// Byte offset of the number in the target file.
    pub position: usize,
    /// The numeric value in the target file.
    pub value: u32,
    /// Whether sibling values at this position form a sequence.
    pub is_sequential: bool,
    /// Number of digits (helps distinguish 3-digit decomposition from absolute).
    pub digit_count: usize,
}
```

New function: `analyze_invariance()` replaces `find_invariant_text()`:
- Aligns number positions across siblings using gap-index matching
- Classifies each number as invariant or variant
- Detects sequential patterns for episode evidence

### Phase 2: Thread `InvarianceReport` into `Pipeline::run_with_context`

**Files:** `src/pipeline/mod.rs`

Change `run_with_context` to:
1. Run Pass 1 on target + siblings (already done)
2. Call `analyze_invariance()` instead of just `find_invariant_text()`
3. Pass `InvarianceReport` into Pass 2

Change `pass2()` signature:
```rust

    &self,
    input: &str,
    all_matches: &mut Vec<MatchSpan>,

```

### Phase 3: Context-aware year disambiguation

**Files:** `src/zone_map.rs`, `src/pipeline/mod.rs`

When `InvarianceReport.year_signals` is available:
- Invariant year-like numbers → classify as title content (override `disambiguate_years`)
- Parenthesized years still win (structural > context)
- Fall back to positional heuristic only when no context

Approach: Add a `year_overrides` field to `ZoneMap` or apply year signals
as a post-Pass1 filter in `run_with_context`, before Pass 2.

### Phase 4: Context-aware episode detection

**Files:** `src/properties/episodes/mod.rs`, `src/pipeline/mod.rs`

When `InvarianceReport.episode_signals` is available:
- Sequential variant numbers → emit as Episode (or Season+Episode via decomposition)
- `is_sequential` + `digit_count == 3` → context validates digit decomposition
- `is_sequential` + `digit_count <= 2` + anime patterns → absolute episode
- No context → `try_digit_decomposition` still runs with `priority = -5`

### Phase 5: Demote heuristics to low-confidence fallbacks

**Files:** `src/properties/episodes/mod.rs`, `src/zone_map.rs`, `src/matcher/span.rs`

- Add `Source` enum to `MatchSpan`: `Structural | Context | Heuristic`
  - Default: `Structural` (zero-cost for existing code)
  - Context signals tag matches as `Source::Context`
  - Heuristic fallbacks tag matches as `Source::Heuristic`
- `try_digit_decomposition`: lower priority to -5, tag as `Source::Heuristic`,
  log with `[HEURISTIC]` prefix. Only runs when no context episode signal.
- `disambiguate_years` "last = year" rule: unchanged behavior but tag as
  `Source::Heuristic`. Context year signals override with `Source::Context`.
- `compute_confidence`: if any result property is `Source::Heuristic` and
  nothing is `Source::Context`, cap at `Confidence::Low`

### Phase 6: Tests

**Files:** `tests/matching_constraints.rs`, `src/pipeline/context.rs` (unit tests)

- Unit tests for `analyze_invariance()` with:
  - Year-as-title siblings (2001.A.Space.Odyssey)
  - Sequential episode siblings (501, 502, 503)
  - Anime absolute episode siblings
  - Mixed content (no clear pattern)
- Integration tests via `hunch_with_context()` for:

## Debugging Story

The main concern with heuristics is **debuggability** — when something goes wrong,
you need to know whether the result came from structure, context, or a guess.

Every heuristic match will:
1. **Log clearly** at `trace` level with a `[HEURISTIC]` prefix
2. **Carry a `source` tag** on the MatchSpan: `Source::Structural`, `Source::Context`, or `Source::Heuristic`
3. **Affect confidence**: heuristic-only results cap at `Confidence::Low`

```
# Structural (no label needed — this is the happy path):
trace: resolved: Season=5 at 10..13

# Context-confirmed:
trace: [CONTEXT] Season=5 at 10..13 (confirmed by 3 siblings)

# Heuristic fallback:
trace: [HEURISTIC] Season=5 at 10..13 (digit decomposition, no context available)
```

This means `hunch -v` instantly tells you *why* a value was chosen.

## What Doesn't Change

- Single-file parsing (`hunch()`) — identical behavior, heuristics still run
- Parenthesized years `(2019)` — always structural, context doesn't override
- SxxExx / NxN patterns — always structural, Groups 1-4 untouched
- TOML rule matching — unchanged
- Title extraction from invariant text — same algorithm, just bundled into InvarianceReport

## Risk Mitigation

- All changes are additive (context enhances, never overrides structural matches)
- Heuristics become fallbacks, not deleted — zero regression risk for single-file
- InvarianceReport fields are all `Option`/`Vec` — graceful degradation
- Compatibility report must stay ≥ 81.8% (run before every merge)
